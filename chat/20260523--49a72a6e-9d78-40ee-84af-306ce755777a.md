## 常见原因与解决方法

### 1. 用法不对（最常见）

该扩展是**代码片段（Snippet）**，不是 Unity 菜单功能。需要：

1. 打开 `.cs` 文件
2. 右下角语言模式为 **C#**（不是 Plain Text）
3. 在**空行或类体内**输入前缀，例如：
   - `MonoBehaviour` → 生成 MonoBehaviour 类模板
   - `Start` → 生成 `Start()` 方法
   - `log` → 生成 `Debug.Log()`
4. 按 **Ctrl+Space** 手动触发补全
5. 在列表里选带 **片段图标**（方块）的项，描述类似 *Unity MonoBehaviour class template*
6. 按 **Enter** 或 **Tab** 插入

> 输入 `MonoBehaviour` 时，C# 补全会先显示**类型** `MonoBehaviour`，片段往往在下面，需要往下找或继续用 Ctrl+Space。

**快速验证**：`Ctrl+Shift+P` → 输入 `Insert Snippet` → 搜索 `MonoBehaviour`，若能看到 Unity 相关片段，说明扩展已加载。

---

### 2. 片段被 C# 补全“淹没”

你安装了 Cursor 自带的 **C#** 扩展（`anysphere.csharp`），类型/方法补全会很多，Unity 片段容易被挤到后面。

建议在用户或工作区设置中加入：

```json
{
  "[csharp]": {
    "editor.snippetSuggestions": "top",
    "editor.suggest.showSnippets": true,
    "editor.tabCompletion": "on"
  }
}
```

设置路径：`%APPDATA%\Cursor\User\settings.json`，或在 Unity 项目的 `.vscode/settings.json` / `.code-workspace` 里配置。

---

### 3. Cursor Tab 干扰 Tab 键

若开启了 **Cursor Tab**（AI 补全），按 Tab 可能接受 AI 建议而不是展开片段占位符。片段插入后若 Tab 跳转 `$0`、`$1` 不正常，可暂时关闭 Cursor Tab，或改用 **Enter** 选中片段。

---

### 4. 修改扩展配置后未重启

若在设置里改过 **Unity Code Snippets**（缩进风格、自动补全类别等），扩展会提示 **Restart VSCode**。需执行 **Developer: Reload Window** 或重启 Cursor 后片段才会更新。

---

### 5. 与 Unity Tools 扩展的区别

你还安装了 `zlorn.vstuc`（Unity Tools），其中的 **Message Completion** 是另一套 Unity 生命周期方法补全，和 Kleber 的 Snippets **不是同一个功能**。两套可以并存；若 vstuc 的补全也不工作，需要在 Unity 里 **Regenerate project files** 并确保 C# 语言服务正常。

|------|------|
| `editor.snippetSuggestions: "top"` | Unity 片段排在补全列表最前 |

| `editor.tabCompletion: "on"` | 可用 Tab 接受片段 |
| `editor.quickSuggestions` | 在代码区自动弹出补全 |
| `files.associations` | 确保 `.cs` 识别为 C# |
