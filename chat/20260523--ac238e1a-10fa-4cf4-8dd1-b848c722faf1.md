## 3. 两者如何串联

```mermaid
flowchart LR
    A[相机屏幕中心射线] --> B[计算准心落点]
    B --> C[Root 水平转向]
    C --> D[Muzzle 随 Root 旋转]
    D --> E["firePoint.forward"]
    E --> F[射击射线 + 命中判定]
```

同一帧内大致顺序（`Update`）：

1. `HandleMovement` → `HandleRootRotation`：Root 转向准心
2. `HandleFire` → `TryFire` → 从 **已更新后的 Muzzle** 发射射线

因此：**Muzzle 不决定“瞄哪”，只决定“从哪、朝哪打出去”**；瞄哪由相机射线 + Root 旋转间接决定。

## 4. 需要注意的差异（第三人称常见）

| 项目 | 瞄准射线 | 射击射线 |
|------|----------|----------|
| 起点 | 相机 | Muzzle |
| 方向 | 相机 forward | Muzzle forward |
| 俯仰 | 相机有 pitch | Root 只水平转，Muzzle 无独立俯仰 |

结果是：

- **准心**（屏幕中心）和 **枪口射线** 可能不完全重合
- Muzzle 在 Root 上有偏移 `(0, 0.44, 0.5)`，越肩视角下更容易偏
- 扩散在 `firePoint.forward` 上做随机偏转，与相机射线无关

## 5. 总结

- **Muzzle = `firePoint`**：弹道的物理起点和方向基准
- **相机射线**：只控制角色/枪口大致朝向，不参与 `WeaponRuntime` 的命中检测
- **命中判定 / 可视化**：全部从 Muzzle 发出，并排除 Player 自身 Layer/Collider
