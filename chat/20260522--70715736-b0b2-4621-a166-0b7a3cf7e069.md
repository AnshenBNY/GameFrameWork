## 为什么找不到

根据 [Cursor 官方安全文档](https://cursor.com/docs/agent/security)：

> Cursor 支持工作区信任（Workspace Trust），但**默认关闭**。开启后，打开新工作区时会提示选择「正常模式」或「受限模式」。

你当前的 `settings.json` 里**没有**开启该功能：

```json
"security.workspace.trust.enabled": true  // ← 你没有这项
```

因此：

- 命令面板里搜不到 `Workspace: Manage Workspace Trust` /「管理工作区信任」
- 状态栏也不会出现「Restricted Mode / 受限模式」
- **当前工作区在功能上等同于「已受信任、无限制」**

## 对你当前情况意味着什么

排查 Hook 不运行时，**可以跳过「确认工作区为 Trusted」这一步**。
在 Cursor 默认配置下，Hook 不会被「受限模式」拦截；若 Hook 仍不工作，原因更可能在别处，例如：

- Hook 脚本路径或 Node 执行方式
- `hooks.json` 配置
- Cursor 的 Hooks 输出通道 / 日志

## 若你仍想手动开启该功能（可选）

1. `Ctrl+Shift+P` → **Preferences: Open User Settings (JSON)**
2. 添加：

```json
"security.workspace.trust.enabled": true
```

3. 重启 Cursor
4. 再次打开项目时会出现信任提示；在命令面板中即可搜到 **Workspace: Manage Workspace Trust**

注意：开启后，未信任的工作区会进入**受限模式**，部分 AI 功能会受限，一般**不建议**仅为排查 Hook 而开启。
