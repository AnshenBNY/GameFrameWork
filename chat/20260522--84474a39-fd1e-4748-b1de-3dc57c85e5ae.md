## 根本原因

日志中的实际错误是：

```text
{"errno":-4048,"code":"EPERM","syscall":"spawn"}
Error: spawn EPERM
```

含义：Cursor 尝试启动 `powershell.exe`、`node.exe` 等子进程时，被系统以 **权限不足（EPERM）** 拦截。

Shell 路径本身是正确的：

```text
C:\WINDOWS\System32\WindowsPowerShell\v1.0\powershell.exe
```

今天约 **14:09** 起开始出现该错误（上午 09:28 的会话正常），说明更像是 **安全软件拦截** 或 **Cursor 更新后的沙箱行为变化**，而不是项目配置问题。

该问题同时会影响：

- 集成终端（你看到的「终端进程启动失败」）
- Agent 执行命令
- `.cursor/hooks` 里的 `node` 脚本（你的 `sync-transcript` Hook 也报了同样的 `spawn EPERM`）

---

## 解决步骤（按优先级）

### 1. 启用 Legacy Terminal Tool（Cursor 官方 workaround）

1. 打开 **Cursor Settings**（`Ctrl + ,`）
2. 进入 **Agents → Terminal**
3. 开启 **「Legacy Terminal Tool」**（旧版终端工具）
4. **完全退出并重启 Cursor**（不是只关窗口）

这会绕过 Agent 终端沙箱，多数 Windows 上的 `spawn EPERM` 可由此解决。参考：[Cursor 论坛讨论](https://forum.cursor.com/t/cursor-agent-terminal-spawn-eperm-on-windows/150268)

---

### 2. 将 Cursor 加入 Windows 安全软件白名单

**Windows 安全中心：**

1. **病毒和威胁防护 → 管理设置 → 排除项**
2. 添加以下路径为排除项：
   - `C:\Users\Administrator\AppData\Local\Programs\cursor\`
   - `C:\Users\Administrator\AppData\Roaming\Cursor\`

**若开启了「勒索软件防护 / 受控文件夹访问」：**

- **勒索软件防护 → 允许应用访问受控文件夹**
- 将 **Cursor.exe** 加入允许列表

有第三方杀毒（360、火绒、卡巴斯基等）时，同样需要把 Cursor 和 `node.exe` 加入信任/白名单。

---

### 3. 在 Cursor 外验证系统终端是否正常

在 **Windows 自带的「终端」或 PowerShell」**（不是 Cursor 内置终端）中执行：

C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -NoLogo -Command "Write-Host OK"
